# Implementation Plan: Monobank Bank Provider Adapter

**Branch**: `007-monobank-adapter` | **Date**: 2026-04-19 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/007-monobank-adapter/spec.md`

## Summary

Add Monobank (Ukraine) as a second bank provider alongside Plaid. The feature introduces the `IBankProvider` domain interface (fulfilling Constitution Principle I), implements `MonobankAdapter` calling the Monobank personal API, adds a `MonobankCredential` entity for encrypted token storage, and extends the connect/sync/disconnect flows to be provider-aware. The frontend Connect Account page gains a provider selection step.

## Technical Context

**Language/Version**: C# 13 / .NET 9 (backend) · TypeScript 5.x strict / Angular 20 (frontend)  
**Primary Dependencies**: ASP.NET Core 9, EF Core 9, MediatR, Hangfire, `System.Net.Http` (no new NuGet packages required — Monobank API is plain REST)  
**Storage**: PostgreSQL 14 — existing `BankSyncDbContext`; migration M002 adds `MonobankCredentials` table and modifies `BankAccounts`  
**Testing**: xUnit (backend unit + contract tests) · Vitest + Playwright (frontend)  
**Target Platform**: Linux server (Docker), existing Docker Compose stack  
**Project Type**: Web service (modular monolith extension)  
**Performance Goals**: 90-day initial import ≤ 2 minutes for single-account users (3 × 31-day windows with mandatory 60s rate-limit waits)  
**Constraints**: Monobank personal API rate limit: 1 req/60s per endpoint; no new NuGet packages; existing AES-256-GCM encryption reused  
**Scale/Scope**: Single developer; personal finance app; ~1–5 Monobank accounts per user
